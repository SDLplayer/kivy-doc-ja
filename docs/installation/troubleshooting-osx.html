Troubleshooting on OS X(翻訳済み)¶

OS XにKivyをインストールするのに問題がありますか？ このページには問題があります

“Unable to find any valuable Window provider” エラー¶

[DEBUG  ] [Cache       ] register <kv.image> with limit=None, timeout=60s
[DEBUG  ] [Cache       ] register <kv.atlas> with limit=None, timeout=Nones
[INFO   ] [Image       ] Providers: img_imageio, img_tex, img_dds, img_pil, img_gif (img_pygame ignored)

[DEBUG  ] [App         ] Loading kv <./pong.kv>

Kivyは何らかの理由でPyGameをインポートできない可能性が最も高いです。次のセクションに進みます。

Check for Problems with Your PyGame Installation(PyGameのインストールの問題を確認する)¶

最初に、PyGameが動作していることを確認してください。

インタラクティブなPythonインタープリタを起動し、pygameをimportしてみましょう：：

$ python
Python 2.7.3 (v2.7.3:70274d53c1dd, Apr  9 2012, 20:52:43)
[GCC 4.2.1 (Apple Inc. build 5666) (dot 3)] on darwin
Type "help", "copyright", "credits" or "license" for more information.
Python 2.7.3 (v2.7.3:70274d53c1dd, Apr  9 2012, 20:52:43)
Type "copyright", "credits" or "license" for more information.
>>> import pygame

pygameを問題なくインポートできたら、次のセクションに進んでください。

しかし、エラーが発生した場合、PyGameは正常に機能していません。

これらのPyGameインポートエラーを解決する最も簡単な方法は次のとおりです：

  1. ``pygame``パッケージを削除します。 （たとえば、上記のエラーが発生した場合は、

    delete /Library/Frameworks/Python.framework/Versions/2.7/lib/python2.7/site-packages/pygame/ とそれに伴うeggを削除してください。

  2. ご使用のOS Xのバージョン用のPyGameバイナリをインストールしてみてください。ダウンロードは下から

    http://www.pygame.org/download.shtml. からダウンロードしてください。

  3. この方法を繰り返し、動作するPyGame OS Xバイナリを探してください。
