Google Summer of Code - 2016(翻訳済み)¶

Introduction(イントロダクション)¶

Kivyは、マルチタッチアプリケーションなどの革新的なユーザーインターフェイスを使用するアプリケーションの迅速な開発のための、クロスプラットフォームでビジネスフレンドリーなGPUアクセラレーションなオープンソースなPythonライブラリです

Kivyオーガナイザーはいくつかの主要プロジェクトを見ています：

  • Kivy GUIライブラリ

  • Python-For-Android コンパイルツール

  • Kivy-iOS コンパイルツール

  • PyJNIus PythonからJavaにインタフェースするためのライブラリ。

  • PyOBJus PythonからObjective-Cにインタフェースするためのライブラリ。

  • Plyer プラットフォーム依存のAPI用のプラットフォームに依存しないPythonのラッパー。

  • Buildozer Android、iOS、およびデスクトップ用の一般的なPythonパッケージを作るツール。

  • KivEnt - 大量の動的ビジュアルデータを処理する最適化された方法を提供する2dゲームエンジン。

  • Kivy Designer - Kivy用のグラフィカルなGUIデザイナー。

これらのプロジェクトにより、ユーザーはネイティブAPIを使用する主要なオペレーティングシステムごとにアプリケーションを作成できます。私たちの目標は、どこでも同じコードベースで動作するPythonアプリケーションの開発を可能にし、特定のオペレーティングシステムのユーザーが期待しているプラットフォームに依存するAPIと機能を利用することです。

どのプロジェクトを選択するかに応じて、Pythonに加えてCython、OpenGL ES2、Java、Objective-C、Cも知っておく必要があります。私たちは計算とグラフィックスのパフォーマンスを重視してCythonとOpenGLを多用し、他の言語は通常OSやプロバイダレベルのAPIにアクセスします。

Google Summer of Code 2016に参加したいと考えています。このページには、GSoCプロジェクトのアイデアとKivyフレームワークに貢献する学生のためのガイドラインが掲載されています。

(要件)¶

入学した学生はここで強調されているいくつかの基本的な要件を満たしていると想定しています：

  • Pythonの中級レベルの知識。

  • gitとgithubを快適に使いこなす（Kivyとその姉妹プロジェクトはすべてgithubで管理されています）。githubを使ったことがない人は興味深い`チュートリアル <https://guides.github.com/activities/hello-world/>`_があります。

  • イベントドリブンプログラミングで快適にできる。

  • Kivyと取り組む予定の姉妹プロジェクトに適したツール/環境があります。 たとえば、PyOBJusで作業できるようにするには、PyJNIusで動作するにはAndroidデバイスが必要なように、iOSデバイス、Xcodeと開発者ライセンスを持つOS Xにアクセスする必要があります。また、plyerで作業するにはハードウェアへのアクセスが必要な両方のプラットフォームが必要になります

追加のスキルが特定のプロジェクトにリストされることがあります。

contribution guide に慣れてくださいスピードアップをお手伝いしますが、事前に能力を証明している生徒には優先されます。

開始方法¶

Kivyの場合、最も簡単な方法は、特定のプラットフォーム用の開発バージョンのインストール手順に従うことです。

http://kivy.org/docs/installation/installation.html#development-version

それ以外の場合は、gitから該当するプロジェクトをインストールしてPYTHONPATHに追加すれば十分です。

たとえば.. PyJNIusの場合：

git clone http://github.com/kivy/pyjnius
export PYTHONPATH=/path/to/pyjnius:$PYTHONPATH

Project Ideas(プロジェクトのアイデア)¶

Kivy開発チームが提供している将来性があるあるアイデアがあります。興味のあるプロジェクトがない場合は、＃kivy-devのプロジェクトアイデアについてお話してください。

初心者のプロジェクト¶

これらのプロジェクトは、大学レベルのPythonに精通していて、プラットフォームの特質についてほとんど知識が必要ない人に適していますす。

中級のプロジェクト¶

これらのプロジェクトには、いくつかのOSレベルの詳細、いくつかのOpenGLのやりとり、または平均的なPythonistaのwheelhouseから少し外れている可能性のある他のトピックに関する、大まかな知識が必要になる場合があります。

Plyer:

説明：

Plyerは、Kivyがサポートするデスクトップとモバイルプラットフォームで一般的にある機能を使用する、プラットフォームに依存しないPython APIです。デスクトップまたはモバイルデバイスの機能にアクセスするための安定したAPIをユーザーに提供します。

学生は、現在p4aプロジェクトにあるいくつかの「.java」コードをPlyerのより適切な場所に置き換えます。また、アクセス可能性、Bluetooth Low Energy、連絡先へのアクセスと編集、共有、NFC、アプリ内ブラウザ、Wi-Fi（Wi-Fiへのアクセスの有効化、無効化、アクセス権の設定など）を含むPlyerによるプラットフォーム固有の機能へのアクセスを改善します。カメラのキャプチャ（ビデオ）、カメラの表示、Google Playの統合、電話の呼び出しインターフェイス、SMSインターフェイス、ジオロケーション、通知とのやりとり、国際化（I18N）、Wi-Fiサービス（Wi-Fiダイレクト、ネットワークアクセス性、ネットワーク上の現在のIP情報など） ）、および欠落しているプラットフォームの実装をすべて既存の機能から削除します

フードの下ではAndroid上ではPyJNIus、OS XとiOSではPyOBJus、Windowsではctypes、LinuxではネイティブAPIを使用します。これにはおそらくPyOBJusとPyJNIusを改良して、現在はできないインタフェースを処理することも含まれています。

参考文献:
  • https://github.com/kivy/plyer
  • https://github.com/kivy/pyjnius
  • https://github.com/kivy/pyobjus
  • https://github.com/kivy/python-for-android
  • https://github.com/kivy/kivy-ios
期待される成果：

成果には、p4a/kivy-iosからJava/PyOBJusコードをplyerに移動し、新しいファサードの一部または全部を学生に決定させることが含まれます。

  • メンター：Akshay Arora、Ryan Pessa

  • 要件：Linux、Windows、OS X、iOSデバイス、Androidデバイスへのアクセス。

  • タスクレベル：中級

  • 望まれるスキル：PyJNIus、PyOBJusに精通している

高度なプロジェクト¶

これらのプロジェクトには、Kivyの既存の内部構造、クロスプラットフォームコンパイルの細部、またはかなり高度なトピックに関する非常に深い知識が必要です。あなたがPythonの内部に慣れていれば、Cコードを使って作業し、Cythonを使って独自のC拡張を構築すれば、これらのプロジェクトはアピールになるかもしれません。

Kivent：タイル型統合

説明：

KivEntは、Kivyの上に構築されたモジュラーエンティティコンポーネントベースのゲームエンジンです。 KivEntは、特殊なCython構造を使用したPythonの最悪のオーバーヘッドのいくつかを回避する、Pythonでゲームを構築するための高いパフォーマンスのアプローチを提供します。

タイル張りのサポート：16進、正方形、および等角四角形のタイルがサポートされています。この作業には、ロジックゲームシステムとレンダリングゲームシステムの両方を書くことが含まれます。さらに、学生はKivEntで使用するためのMITライセンスtmx（タイルファイルフォーマット）ローダーの作成を担当します。

参考文献:
  • http://www.mapeditor.org/
  • https://github.com/kivy/kivent
期待される成果：

成果には、KivEntゲームエンジン用にリリースされた新しいkivent_tiledモジュールが含まれます。

  • メンター：Jacob Kovac, Gabriel Pettier

  • 要件：少なくとも1つのKivyプラットフォームへのアクセス。

  • タスクレベル：上級

  • 望まれるスキル：Cython、Python、およびゲーム開発関連の数学の概念に精通してる。

説明：

Python for Androidは、あなたが望むモジュールを含む独自のPythonディストリビューションを作成し、Python、libs、およびあなたのアプリケーションを含むAPKを作成するプロジェクトです。

How to Contact devs(devsに連絡する方法)¶

すべてのコミュニケーションはパブリックチャンネルを介して行われなければなりません。プライベートメールやIRCメッセージは推奨されません。

Kivyユーザーフォーラムhttps://groups.google.com/group/kivy-usersで質問するか、kivy-users@googlegroups.comにメールを送信してくださ

kivy-devユーザーグループにも参加してください:https://groups.google.com/forum/#!forum/kivy-dev

上記の開発者のIRCハンドルを取得するには、https://kivy.org/#aboutusにアクセスしてIRC（オンラインチャット）にお問い合わせください。

接続する前に`IRCルール <https://kivy.org/docs/contact.html>`_ を必ずお読みください。 `Webchatに接続 <http://webchat.freenode.net/?nick=kvuser_GSOC_.&channels=kivy&uio=d4>`_します。

開発者のほとんどはヨーロッパ、インド、北米にいますすので、これらの地域の典型的な起床時間を覚えておいてください。

How to be a good student(どうしたら良い生徒になれるか)¶

学生として参加して受け入れられる可能性を最大限にしたい場合は、今中に私たちと話をして、ワークフローに慣れるためにいくつかの小さな問題を修正してみてください。私たちとうまくやっていけることが分かっているなら、選ばれる機会ははるかに多くなるでしょう。

ここにチェックリストがあります：

  • ウェブサイトを読んで、少なくともドキュメントを見てください。

  • ソースコードを見てください。

  • 寄付のガイドラインを読んでください。

  • 貢献してください！ Kivyは開発プロセスにどのように取り組んでいるのかを見たいです。興味のあるKivyプロジェクトの課題トラッカーを見て、プルリクエストを送信してください。簡単なバグや文書の変更が可能です。私たちはあなたの能力を評価するのではなく、仕事の仕方を見たいです。心配しないで作業を選択してください。

  • アイデアリストから興味深いと思うアイデアを選ぶか、自分のアイデアを思いついてください。

  • いくつかは「自分」で研究してください。 GSoCは、片側の相互作用だけでなく、意見を交換に関するものです。私たちのサポートで合意した目標を達成しようとするのはあなたです。その際の主な推進力は自分自身であるべきなのは明らかです。多くの学生がポップアップし、何をすべきか尋ねます。その決定は、あなたの興味とスキルに基づいているべきです。あなたが真剣であり、イニシアチブをとることを私たちに示してください。

  • したいことについての`プロポーサル <https://wiki.python.org/moin/SummerOfCode/ApplicationTemplate2016>`を書きます。プロジェクトの現在の状態を理解し、改善したいこと、方法などを含めます

  • 適時、私たちと提案について話し合います。フィードバックを得ます。

  • 我慢して！特にIRCで。私たちが利用できるなら、私たちはIRCでコンタクトします。そうでない場合は、電子メールを送信して待ってください。ほとんどの質問はすでにドキュメントやその他の場所で回答されており、調査で見つけられます。あなたの質問には、あなたが実際に何を求めているのかを考えて、いくつかの初歩的な研究をしたことが反映されているはずです。

  • 何よりも、楽しい時間を過ごし、コミュニティと交流することを忘れないでください。コミュニティはコード自体と同じくらいオープンソースの一部です。

What to expect if you are chosen(選ばれた場合に期待すること)¶

  • すべての生徒は毎日#kivyチャンネルと＃kivy-dev ircチャンネルに参加する必要があります。これは開発チームが内部的にもユーザーとも通信する方法です。

  • あなたと指導者は、夏の間の2週間のマイルストーンに同意します。

  • 開発は、Kivyのmasterブランチのフォークで行われます。少なくとも週に1つのPRを週にブランチに提出することをお勧めします。これは進行状況を報告し、作業の過程で発生した苦労する可能性のあることを文書化するためのフォーラムになります。

  • 2週間ごとのPRまたは2つのマイルストーンがない場合、納得のいく状況がなければ失敗に終わります。 何かが起きた場合はできるだけ早くメンターにお知らせください。 マイルストーンが届かない場合、私たちはあなたと一緒に目標を再評価します。

  • プロジェクトが完了したら変更内容がマスターに統合され、関連するすべてのプラットフォームで徹底的にテストされています。
